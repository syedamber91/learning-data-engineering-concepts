*What Alex understood:* So an RDD is Spark's most basic piece of data, and I can fully describe one with five things — its partitions (the pieces), a function to compute each piece, its dependencies (what it came from), an optional partitioner just for key-value ones, and optional preferred locations. It never changes once created. And it's lazy: transformations just stack up a DAG plan, and only an action makes Spark actually run everything.

## Follow-up questions

**Alex:** If transformations don't run anything, what actually makes Spark do the work?

**vutr:** An action makes Spark do the work. The note says RDDs are lazily evaluated — transformations only build up a DAG, and it's the actions that actually trigger execution.

**Alex:** Why is the partitioner only 'optional' and tied to key-value RDDs?

**vutr:** The note lists the partitioner as an optional partitioner for key-value RDDs, so it's one of the five properties but only applies to key-value RDDs; the note doesn't explain further why it's optional beyond that.
