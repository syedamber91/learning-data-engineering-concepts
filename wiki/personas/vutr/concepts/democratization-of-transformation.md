Data transformation has been democratizing. Logic that used to require engineers with robust coding skills can now be written by a data analyst or analytics engineer using plain SQL — [[dbt]] is the tool that made this possible. Crucially, dbt does not replace data engineers; it enables data engineers (DEs) and data analysts (DAs) to collaborate, letting SQL-literate analysts contribute transformations while DEs still own the harder plumbing. The open question is how far this goes before the absence of a real data model starts to hurt.
