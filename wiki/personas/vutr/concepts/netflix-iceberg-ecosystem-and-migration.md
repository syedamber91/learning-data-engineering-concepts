Netflix's data platform is cloud-native and built around storage-compute separation: source systems (Cassandra, RDS, CockroachDB) feed a Kafka-and-Flink stream-processing layer handling trillions of daily events, which lands in a lakehouse backed by S3 and Apache Iceberg; a metadata service called Metacat sits over that lakehouse giving every table a single three-part name regardless of whether it's actually Hive or Iceberg underneath, so consumers never need to know or care; and a compute layer of Trino (interactive), Spark (pipelines), Druid (real-time dashboards), and Elasticsearch (search) runs on Netflix's own Titus container platform over EC2, ultimately serving Tableau and Jupyter at the top. Iceberg itself exists because Hive stopped being tenable at Netflix's actual scale: 600,000 Hive tables and 250 million partitions, with no ACID transactions, no atomicity, no time travel or rollback, and full-table rewrites required for something as routine as changing a partitioning scheme or updating a schema.

Getting Iceberg into production at that scale required Netflix to build an entire operational ecosystem around the format, not just adopt it:

- **Polaris**, Netflix's own Iceberg catalog. Netflix initially ran both Hive and Iceberg tables through the same Hive Metastore backed by RDS MySQL, but Hive's pessimistic-locking commit pattern and Iceberg's optimistic-concurrency commit pattern put irreconcilable pressure on the same RDS instance. Polaris is a metastore purpose-built for Iceberg, backed by CockroachDB specifically for its horizontal scalability, and it implements the Iceberg REST catalog specification — meaning any third-party engine that speaks that standard connects through one endpoint, a real improvement over Hive Metastore's more bespoke integration story.
- **Janitors**, Netflix's cleanup service, running in three configurable modes: a TTL Janitor removing data past a set expiration, a Snapshot Janitor pruning old Iceberg snapshots on their own TTL, and an Orphaned File Janitor removing files that no Iceberg table references at all (mostly leftovers from failed metadata commits — see [[occ-on-object-storage]]). The pipeline runs Spark jobs to find expired tables, hands that list to Amazon SQS, and an S3 deletion service performs a soft delete (data is hard-deleted only after 5–7 days) while writing to an audit log that lets an accidental table drop be restored.
- **Autotune**, a background compaction service. Write operations naturally produce many small files; Autotune listens for metastore write-events via SQS, decides whether a table needs compacting based on its configured compaction settings, and launches a Spark job to rewrite the data files — covering both plain small-file merging and delete-file compaction for merge-on-read tables (see [[copy-on-write-vs-merge-on-read]]).
- **Autolift**, a cross-region data-localization service. Because most of Netflix's compute runs in US-East-1, Autolift migrates up to three petabytes of data per day into that region using Iceberg's replace-commit API — scanning incoming snapshots for files that need to move, then atomically removing the remote-region files and replacing them with local copies.
- **Secure Iceberg tables.** Netflix stores table-level access-control metadata (which users/ACLs apply) directly inside the Iceberg table itself, treating each table as a distinct resource tied to specific S3 prefixes. When a user queries a table, the Spark driver calls an external signer service Netflix built specifically as the policy-enforcement point: the signer checks the metastore for the user's ACL membership and, if authorized, issues a short-lived AWS STS token that the Spark driver passes down to its executors — so users interact with tables through this signed-credential layer and never touch raw S3 permissions directly.
- **Hive-to-Iceberg migration tooling**, built to move roughly 1.5 million Hive tables with minimal downtime and, critically, minimal data movement: rather than physically copying data, the tooling creates an Iceberg metadata layer directly on top of the existing Hive data files wherever possible (a small minority of tables still required an actual copy). Five components run asynchronously against shared state: the **Processor** wakes on an interval and identifies tables due for migration, extracting ownership metadata; the **Communicator** notifies table owners and downstream consumers of the coming migration; the **Migrator** does the actual work (wrap-in-place or physical copy); the **Reverter** can roll a table back to Hive if something goes wrong, made possible because Netflix keeps the original Hive table in sync with the migrated Iceberg table for a period after migration; and the **Shadow** performs that sync, delta-copying via Iceberg's own metadata layer. Each component can run as multiple parallel instances, with tables distributed across instances by consistent hashing.

Netflix's Write-Audit-Publish data-quality pattern — detailed in [[iceberg-branching-and-wap-implementation]] — runs on top of this same Iceberg ecosystem, using table branching to audit data without a second physical write.
