I received feedback, although my Spark articles bring a lot of value.

However, the lessons are forgotten after a few weeks.

I’ve written more articles that include code.

But switching back and forth between the article and the IDE to follow the instructions and run code is exhausting.

I wondered whether there is a better way to execute the approach, “read enough theory and start running something. “

So I created a small project that allows you to:

* Read the Spark internals theory
* Run and submit the Spark code.
* Verify if things work.
* And, observe the Spark application result.

RIGHT ON YOUR LAPTOP, WITHOUT INTERNET.

It will take you **less than 15 minutes to install** the CLI tool and more than 2 hours to complete all 32 Spark exercises.

It’s like playing a game.

For an exercise, you read the lesson, adjust the code based on the instruction, verify if it works, and move on to the next one.

You will learn nearly every internal concept that makes you ready for a Spark production application. From the journey of the application, RDD basics, transformation vs action, to how Spark executes aggregation and join.
